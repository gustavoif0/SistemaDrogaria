# Skill — Pharma UI system

## Objetivo

Padronizar o design system do PharmaERP.

## Regras visuais

1. Layout desktop-first.
2. Sidebar fixa à esquerda.
3. Topbar fixa no topo da área principal.
4. Cards com borda leve, fundo branco e indicador visual.
5. Tabelas densas, legíveis e com cabeçalho fixo quando útil.
6. Badges de status:
   - Verde: OK, ativo, autorizado, transmitido.
   - Amarelo/laranja: pendente, vencendo, atenção.
   - Vermelho: erro, rejeitado, cancelado, vencido.
   - Cinza: neutro, fechado, inativo.
7. Botões:
   - Primário verde para salvar/finalizar/enviar.
   - Secundário branco/cinza para ações neutras.
   - Perigo vermelho para excluir/cancelar.
8. Formularios complexos devem usar abas.
9. Pesquisa de produto deve usar modal com filtros e tabela.
10. Valores financeiros devem ter destaque e alinhamento à direita.

## Saída esperada

- Componentes reutilizáveis.
- Tokens de cores/espaçamento.
- Estados visuais.
- Exemplos de uso.
