# Skill — Accessibility ERP review

## Objetivo

Melhorar usabilidade e acessibilidade sem descaracterizar o ERP.

## Checklist

1. Todo input deve ter label ou aria-label.
2. Botões devem ter texto claro ou aria-label.
3. Navegação por teclado deve funcionar em menus, abas e modais.
4. Modal deve controlar foco.
5. Tabelas devem ter cabeçalhos semânticos.
6. Cores não devem ser o único indicador de status.
7. Contraste deve ser suficiente.
8. Valores financeiros importantes devem ser legíveis.
9. Alertas críticos devem ter texto explícito.
10. Campos obrigatórios devem ser identificáveis.

## Saída esperada

- Problemas encontrados.
- Correções aplicadas.
- O que precisa de revisão manual.
