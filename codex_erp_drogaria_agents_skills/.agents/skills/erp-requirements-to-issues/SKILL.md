# Skill — ERP requirements to issues

## Objetivo

Converter requisitos funcionais do ERP de Drogaria em backlog claro para desenvolvimento.

## Procedimento

1. Leia `AGENTS.md` e `docs/PROJECT_BLUEPRINT.md`.
2. Identifique o módulo afetado: Produto, Balcão, PDV, Estoque, Financeiro, Fiscal, SNGPC, BI ou Configurações.
3. Separe:
   - Requisito confirmado.
   - Inferência do protótipo.
   - Ideia futura.
4. Gere histórias no formato:
   - Como [perfil], quero [ação], para [benefício].
5. Para cada história, crie critérios de aceite testáveis.
6. Marque prioridade: MVP 1, MVP 2 ou Futuro.
7. Liste dependências técnicas e de dados.

## Saída obrigatória

- Resumo do módulo.
- Histórias de usuário.
- Critérios de aceite.
- Regras de negócio.
- Tarefas técnicas.
- Riscos e dúvidas.
